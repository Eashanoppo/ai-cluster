## 4. PARTICIPATION RULES

### 4.1 Eligibility
- The hackathon is open to **DIU Students and DIU Alumni ONLY**.
- Each team must consist of exactly **3 student members** and **1 mentor** if applicable.
- The mentor must be a DIU faculty member or a DIU alumnus/alumna.
- A participant may not be a member of more than one team.
- Each team must choose one of the three official tracks during registration.

### 4.2 Team Formation
- Team composition may not be changed after the Phase-1 submission deadline.
- All team members must present a valid DIU student/alumni ID at the final round.

### 4.3 Code of Conduct
- **Originality:** All submitted work must be original. Pre-built or previously developed projects (even partial) are not accepted.
- **Fair Play:** Any form of plagiarism, cheating, or misrepresentation will result in immediate disqualification.
- **Respect:** Participants must treat all fellow participants, mentors, judges, organizers and volunteers with respect.
- **Compliance:** Participants must comply with all rules stated in this rulebook and any announcements made by organizers.

> ⚠️ **NOTE:** The organizing committee reserves the right to make final decisions on all matters. Participants will be notified of any notable rule modifications.

## 5. HACKATHON TRACKS
This hackathon focuses on a single combined track: **AI for Cluster Intelligence**. Teams will build AI-powered tools and systems that make GPU and CPU clusters faster, cheaper and more reliable. Rather than choosing between separate tracks, all teams tackle the same problem space and may approach it from one angle or combine multiple angles in their solution.

**Track: AI for Cluster Intelligence**  
Teams may tackle any of the following angles or combine them into a single solution. The goal is to build AI-driven tools that help compute clusters run smarter.

| Angle | Example Projects | Relevant Technologies |
| :--- | :--- | :--- |
| **Smart Scheduling** | ML-driven workload placement and GPU bin-packing optimizer | Kubernetes, Ray, NVIDIA DCGM, Python |
| **Predictive Operations** | Anomaly and failure-prediction system using cluster telemetry | Prometheus, NVIDIA DCGM, Python, ML anomaly detection |
| **Cost Optimization** | Automated idle GPU detection and cost-reduction dashboard | Kubernetes, NVIDIA DCGM, Prometheus, Python |

## 9. PROJECT RULES & SUBMISSION

### 9.1 Phase-1 Submission Rules
- All submissions must be made through the official hackathon website before the stated deadline.
- Required items: Concept Note, Presentation (max 10 slides), 3–5 minute video and Implementation/ Technical Analysis.
- Submissions must correspond to the chosen track. Off-track submissions will be disqualified.
- Teams may update their submission before the deadline, the latest version will be evaluated.

### 9.2 Phase-2 (Final) Project Rules
- **Original Work:** All code and materials must be developed during the hackathon day. Pre-built or previously developed work (even partially) is strictly prohibited.
- **GitHub Repository:** Teams must create a new GitHub repository at the start of the sprint and submit the link at end of the competition.
- **No Pre-written Code:** Existing personal libraries or private repositories developed before the event are not permitted as the core submission.
- **Open Source Libraries:** Using publicly available open-source packages, frameworks and APIs is allowed and encouraged.
- **AI Assistance:** AI coding assistants (e.g., GitHub Copilot, ChatGPT, Claude) may be used for suggestions but must not generate the entire solution.

### 9.3 Disqualification Criteria
- Submitting pre-built or plagiarized work
- Team member substitution after the Phase-1 deadline
- Misbehavior, disrespect, or violation of any rule in this rulebook
- Failure to pay Phase-2 participation fee by the stated deadline (for finalists)
- Incomplete Phase-1 submission
